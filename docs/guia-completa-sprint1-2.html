1.2 Seguridad y Autenticación

Rate limiting de login

Problema: El rate limit apuntaba a pathname === '/' pero NextAuth v5 POSTea a /api/auth/callback/credentials.

Solución:

  // Antes
  if (pathname === '/' && request.method === 'POST')
  // Después
  if (pathname.startsWith('/api/auth/') && request.method === 'POST')

Autenticación en API pacientes

Problema: GET/PATCH /api/pacientes/[id] no verificaban sesión.

Solución: Nuevo helper requireAuth() que retorna 401 si no hay sesión. GET retorna 404 si no existe. PATCH valida body vacío. Errores internos ocultan detalle en producción.

User enumeration

Problema: Se revelaba si el email existía y cuántos intentos quedaban.

Solución: Mensaje unificado "Email o contraseña incorrectos" en todos los casos. Sin minutos restantes en cuenta bloqueada.

Validación de firma Twilio

Problema: JSON bodies no poblaban params. Firma se validaba contra objeto vacío.

Solución: Flujo validateTwilioRequest() con split producción/desarrollo. JSON bodies pasan params. Error 403 sin detalles en producción. Eliminado console.log con teléfono de paciente.

JSON writes atómicos

Problema: writeJSON() escribía directo al archivo. Race condition en writes concurrentes.

Solución:

  function writeJSON(path, data) {
    const tmp = path + '.tmp.' + process.pid;
    fs.writeFileSync(tmp, JSON.stringify(data), 'utf-8');
    fs.renameSync(tmp, path); // Atómico en el mismo filesystem
  }

1.4 Workflows n8n

WF-01: SQL Injection

Problema: Queries concatenaban strings SQL desde expresiones n8n.

Solución:

  // Antes
  "query": "={{ 'SELECT ... WHERE id = ' + $json.id }}"
  // Después
  "query": "SELECT ... WHERE id = $1",
  "queryParams": "={{ [$json.id] }}"

Además, función esc() mejorada para backslashes, null bytes y caracteres de control.

WF-02: Nodo huérfano

Problema: El nodo "PG - Crear Turno" NO estaba conectado al flujo. Los turnos nunca se guardaban.

Solución:

  // Antes
  Responder Horarios → Confirmación Turno
  // Después
  Responder Horarios → PG Crear Turno → Confirmación Turno

WF-05: Merge node

Problema: 4 queries paralelas alimentaban a Ollama sin Merge. Se ejecutaba 4 veces.

Solución:

  Cron → [PG-Turnos, PG-Pacientes, PG-Mensajes, PG-Recetas]
       ↓
  Merge (combine multiplex)
       ↓
  Code - Agrupar Datos (combina en objeto único)
       ↓
  Ollama - Generar Resumen (UNA ejecución)
